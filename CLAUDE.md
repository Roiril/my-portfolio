## 出力ルール

- 指示がない限り、ファイル全文や長いログを貼らない
- 変更は unified diff（差分）だけを出す
- コマンド出力は要点を3〜7行で要約する

## 進め方

- 1ステップは小さく（原則3ファイル以内）
- 機能追加と整理を混ぜない（整理は別コミット）
- 迷ったら大きく作り直さず、最小変更を優先する

## Git（巻き戻し前提）

- 作業は必ずブランチで行う（例: work/<topic>）
- 作業前に `git status` を確認し、汚れていたら理由を明記する
- 論理的に区切れたら小さくコミットする
- 共有ブランチでは `reset` より `revert` を優先する
- 破壊的コマンドは勝手に実行しない（rm -rf / git clean -fd / git reset --hard）
  - 必要なら事前に「実行してよいか」を質問する
